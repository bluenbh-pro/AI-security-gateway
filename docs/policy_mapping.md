# 프롬프트 유형별 정책 위반 매핑

## 📌 개요

각 프롬프트 유형에서 발생할 수 있는 법령/정책 위반을 상세히 분석하고, 
Agent 3가 이를 탐지하기 위한 **핵심 판단 포인트**를 정의합니다.

---

## 🔹 Type 1: 파일 첨부 + 간단 요청

### 위반 가능 법령

#### 1️⃣ **영업비밀법 (부정경쟁방지 및 영업비밀보호에 관한 법률)**
- **위반 조항**: 제2조 (영업비밀의 정의), 제10조 (영업비밀 침해)
- **위반 내용**: 극비/대외비 문서를 외부 LLM에 제공하면 보안 관리 상태가 이미 침해됨
- **처벌**: 최대 10년 징역 또는 1억원 이하 벌금

#### 2️⃣ **개인정보보호법**
- **위반 조항**: 제15조 (개인정보의 수집 제한), 제17조 (개인정보 이용)
- **위반 내용**: 문서 내 개인정보(직원 이름, 이메일 등)가 포함될 수 있음
- **처벌**: 최대 3년 징역 또는 3,000만원 이하 벌금

### Agent 3 탐지 로직

```python
VIOLATIONS["Type_1"] = {
    "trigger_keywords": [
        "극비", "대외비", "confidential", "secret",
        "전략", "계획", "정책", "지침"
    ],
    "danger_file_extensions": [
        ".pdf", ".doc", ".docx", ".xlsx",  # 문서 파일
        ".pptx", ".csv",
        ".jpg", ".png"  # 스크린샷 등
    ],
    "check_points": [
        "파일 첨부 여부",
        "파일 명에 극비/대외비/비밀 포함 여부",
        "요청이 '번역', '요약', '편집' 등 외부 노출 의도",
        "LLM이 응답을 저장/학습할 가능성"
    ],
    "violation_score": 85  # 높음
}
```

### 판단 규칙

```
IF 파일_첨부 AND (극비|대외비) AND (번역|요약|편집|영어로) THEN
    위반 = TRUE
    법령 = ["영업비밀법", "개인정보보호법"]
    위험도 = 85
    조치 = "차단"
ENDIF
```

## 🔹 Type 2: 텍스트 임베딩 + 데이터 분석

### 위반 가능 법령

#### 1️⃣ **신용정보법 (신용정보의 이용 및 보호에 관한 법률)**
- **위반 조항**: 제34조 (신용정보의 제공 제한)
- **위반 내용**: 신용점수, 대출 기록 등을 제3자(외부 LLM)에 제공하는 행위
- **구체적**: "제3자는 신용정보 이용업무를 할 수 없다" → 외부 LLM은 명백히 제3자
- **처벌**: 최대 3년 징역 또는 3,000만원 이하 벌금

#### 2️⃣ **개인정보보호법**
- **위반 조항**: 제17조 (개인정보의 이용 제한), 제18조 (개인정보의 제공 제한)
- **위반 내용**: 소득, 직업, 연락처 등 개인정보를 동의 없이 제3자에 제공
- **처벌**: 최대 3년 징역 또는 3,000만원 이하 벌금

#### 3️⃣ **데이터기본법**
- **위반 조항**: 제8조 (데이터 이용 원칙)
- **위반 내용**: 개인정보 보호를 무시한 무분별한 데이터 분석
- **처벌**: 행정 처분, 시정 명령
