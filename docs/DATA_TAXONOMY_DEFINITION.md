# 데이터 민감도 7단계 분류 체계

## 개요
금융/보험 기업 임직원(및 외주인력)의 데이터 접근 요청을 평가하기 위한 데이터 민감도 분류 체계.

각 단계는 **기본 점수**, **포함되는 정보**로 정의됨.
실제 의사결정은 **Agent의 동적 리스크 스코어링 결과**에 따라 결정되므로, 
고정된 의사결정 테이블은 사용하지 않음.

---

## 1. 극비 (Level 1) - 기본점수: 80

### 정의
회사의 생존을 위협할 수 있는 최고 수준의 기밀 정보.
CEO/임원진만 접근 가능. 일반 임직원은 접근 불가.

### 포함되는 정보
- **M&A/인수합병 계획** (미발표)
- **신제품 개발 로드맵**
- **신사업 진출 계획** (미공개)
- **핵심 기술 정보** (특허 미등록)
- **회사 폐쇄/구조조정 계획**
- **기밀 전략 문서**

---

## 2. 고유식별정보 (Level 2) - 기본점수: 80

### 정의 (개인정보보호법 제24조)
**법령에 근거하여 개인을 고유하게 구별하기 위해 부여된 식별정보.**
이 정보만으로도 개인을 고유하게 식별할 수 있음.

### 포함되는 정보 (법정 범위만)
- **주민등록번호** (국내)
- **여권번호**
- **운전면허번호**
- **외국인등록번호**

### 제외 사항 (법적으로 고유식별정보 아님)
기업이나 기관에서 부여하는 번호는 고유식별정보가 아님:
- ❌ 임직원 사원번호 (사내 부여 → 일반 개인정보)
- ❌ 고객 가입자번호 (사내 부여 → 개인신용정보)
- ❌ 계약자번호 (사내 부여 → 개인신용정보)

---

## 3. 신용정보 (Level 3) - 기본점수: 70

### 정의 (신용정보법)
**신용정보 주체를 식별할 수 있는 정보로서 거래내용, 신용도 판단에 활용되는 정보.**

### 포함되는 정보

#### 금융 신용정보
- **신용점수 / 신용등급**
- **대출 이력** (대출액, 금리, 만기, 상환 현황)
- **연체 기록 / 연체액**
- **신용 조회 기록**
- **신용 한도**
- **부도 기록 / 파산 기록**
- **신용 불량 기록**
- **카드 한도 / 사용 기록** (금액 포함)
- **계좌번호 / 카드번호** (금융거래 식별)
- **소득 및 재산 정보**
- **입출금 정보** (계좌 이체내역 등)
- **압류 및 체납정보**

#### 보험 신용정보 (생명보험 특화)
- **계약번호 / 증권번호** (보험 계약 식별)
- **상품종류 및 가입금액**
- **보험료** (금액, 납입 주기)
- **보험계약 체결·해지 내역** 및 일자
- **보험료 납입 이력** (납입여부, 납입액, 연체 여부)
- **보험료 연체 기록 및 금액**
- **보험료 자동이체 등록/해제 기록**
- **보험금 청구 이력** (청구일자, 청구액)
- **보험금 지급 기록** (사망, 만기, 만기환급, 적립금 인출 등)
- **보험료 할인/할증 기록** (건강검진결과, 사고기록 기반)
- **보험 사고 기록** (사고일자, 사고유형)
- **보상 이력** (보상여부, 보상액)
- **신용평가등급** (보험회사 내부 신용도 평가)
- **재계약 이력**

## 4. 민감정보 (Level 4) - 기본점수: 70

### 정의 (개인정보보호법)
**사생활을 현저히 침해할 우려가 있는 정보로 별도의 엄격한 보호가 요구되는 정보.**

### 포함되는 정보

#### 건강·성생활 관련
- **과거/현재의 질병 이력** (기왕력)
- **상해 및 진단명**
- **수술 이력 및 입원 기간**
- **약 처방전 내역 및 조제 내역**
- **진료비 영수증 상의 상병코드**
- **건강검진 결과** (혈압, 혈당, 콜레스테롤, 간수치, BMI, 엑스레이 판독 등)
- **의사 소견서/진단서 내용**
- **장애 등급 및 장애 유형**
- **정신과 상담 및 치료 이력**
- **임신 및 출산 정보**
- **흡연·음주 여부**
- **성적 취향**

#### 사상·신념 관련
- **종교**
- **각종 이데올로기 또는 사상적 경향**

#### 노동조합·정당 가입 관련
- **노동조합·정당의 가입·탈퇴여부**
- **노동조합비 및 회비 납입내역**

#### 정치적 견해
- **정치적 사안에 대한 입장**
- **특정 정당의 지지 여부**

#### 기타 민감정보
- **유전자검사 결과** (유전정보)
- **범죄경력자료** (전과, 범죄기록, 재판기록 등)
- **생체인식정보** (지문, 홍채, 안면인식정보 등)
- **인종·민족**

## 5. 개인정보 (Level 5) - 기본점수: 65

### 정의
**개인을 식별하기 위한 기본 정보.**
이 정보만으로는 개인을 식별하기 어렵지만, 다른 정보와 결합하면 개인 식별 가능.

### 포함되는 정보

#### 인적정보
- **성명**
- **생년월일 / 나이**
- **성별**
- **주소** (집주소, 직장주소)
- **전화번호 / 휴대폰번호**
- **이메일 주소**
- **직업**
- **직장명 / 회사명**
- **사진**
- **가족관계**

#### 인사정보 (임직원)
- **학력**
- **경력**
- **자격증**
- **부서 / 팀**
- **사번** (사내 부여)
- **직급 / 직책**
- **담당업무**
- **상벌기록**
- **직무평가기록**
- **입사 및 퇴사일**

#### IT 식별정보
- **로그인ID**
- **접속IP**
- **쿠키**
- **모바일기기 식별번호** (UUID, MAC주소, IMEI 등)
- **서비스 이용기록**
- **GPS 위치정보**

## 6. 대외비 (Level 6) - 기본점수: 50

### 정의
**회사 내부 운영 정보 중 대외 비공개 정보.**
임직원 간 공유는 가능하지만, 회사 외부로 유출되면 안 됨.

### 포함되는 정보
- **경영 방침 / 운영 계획** (공시 전)
- **조직도** (직원 이름 포함)
- **부서별 실적** (구체적 수치)
- **비용 내역서 및 예산**
- **공급업체 정보**
- **계약 조건** (미공개)
- **내부 정책 / 지침 / 세칙 / 가이드라인**
- **교육 자료** (미공개)
- **각 부서에서 업무목적으로 생성된 산출물**
  - 분석 보고서
  - 전략 문서
  - 업무 매뉴얼
  - 프로세스 문서
  - 성과 평가 자료

## 7. 일반정보 (Level 7) - 기본점수: 30

### 정의
**위의 6가지 민감도에 속하지 않는 모든 정보.**
공개 정보 또는 일반인이 알아도 무방한 정보.

### 포함되는 정보
- **회사 개요** (공개 정보)
- **제품/서비스 설명** (공개 목록)
- **공개 뉴스 / 보도 자료**
- **공개 통계** (공시된 수치)
- **시장 분석 정보** (공개 자료)
- **기술 문서** (공개 표준)
- **일반 지식** (점심메뉴 추천, 길찾기 등)
- **기타 리스크가 없는 모든 정보**

## 프롬프트 유형 구분

### 1. "정의/교육 요청" (Educational Request)
```
"신용정보가 뭐예요?"
"극비로 분류되는 정보는 뭔가요?"
"회사에서 개인정보로 분류하는 건 뭐가 있나요?"
```
→ 일반적으로 **Allow** (지식 전달용)

### 2. "개인정보 접근 요청" (Data Access Request)
```
"OOO고객의 카드번호를 조회해줄 수 있을까?"
"A지점 고객들의 이름과 전화번호 리스트를 주세요"
"임직원 급여 정보를 출력해줄 수 있어?"
```
→ 민감도 + 권한에 따라 **Allow/Conditional/Approval/Block**

### 3. "통계/분석 요청" (Analytics Request)
```
"전체 고객의 신용등급 분포는?" (개인 미식별)
"가장 많은 주문이 일어나는 지역은?" (익명화)
"팀별 평균 직급은?" (통계만)
```
→ 개인정보 제외 시 **Allow** (분석용)

## 8. 회사 AI 정책: 활용 불가 사례 (실제 사례 기반)

### 개인정보는 어떠한 경우에도 외부 AI에 입력할 수 없습니다

#### 활용 불가 사례들

**❌ 사번·고객ID를 단순 치환한 값을 익명정보로 판단**
```
사내에 원본 데이터와 연결할 수 있는 매핑테이블이 존재하면 
언제든 원본 식별이 가능 → 가명정보(=개인정보)에 해당
```

**❌ 고객 상담내용을 요약·번역해달라고 요청**
```
대화 내용에 성명, 연락처, 계약정보 등이 포함되어 있다면,
비록 목적이 요약이나 번역이라 하더라도 개인정보 제공에 해당
```

**❌ 실제 고객 데이터의 일부(샘플 10건)를 AI 분석**
```
입력하는 데이터의 건수가 아무리 적더라도,
개인정보를 외부에 제공하는 행위에 해당하므로 금지
```

**❌ 화면 캡처 이미지를 첨부하여 질의**
```
이미지에 있는 텍스트(성명, 계약번호 등)도 
AI의 글자 인식 기술(OCR)을 통해 그대로 읽히므로,
텍스트를 직접 입력한 것과 동일하게 간주
```

**❌ 문서 제목에만 고객명을 남긴 경우**
```
완벽히 익명화했더라도 문서 제목, 파일명 등 어디에든 
개인정보가 단 하나라도 남아 있다면 해당 파일 전체가 활용 불가
```

## 9. 생성형 AI 입력 전 3초 자가점검

다음 중 하나라도 해당될 경우 **입력하지 마세요**:

### ① 실제 사람에 관한 정보인가?

고객, 임직원, 설계사 등 **실제 사람**과 관련된 정보라면 **입력 금지**

❌ 임직원 사번  
❌ 민원인 정보  
❌ 설계사 코드  

➜ 사번, 설계사 코드도 개인정보에 해당합니다

### ② 실제 계약·거래 정보인가?

실제 고객의 **계약이나 금융거래 정보**라면 **입력 금지**

❌ 보험계약정보  
❌ 보험금 지급정보  
❌ 고객 상담내역  

### ③ 특정 사람을 알아볼 수 있는가?

이름을 지웠더라도 **특정 개인을 알아볼 수 있으면 입력 금지**

❌ 서울 서초구 거주 / 53세 / 남성 / 의사 / 위암 진단 / 2026년 보험금 3억원 청구

➜ 이 정보를 AI에 입력해도 괜찮을지 **조금이라도 망설여진다면 입력하지 마세요**

## 주의사항

**실제 의사결정은 Agent의 동적 리스크 스코어링 기반입니다.**

이 분류 체계는 데이터 유형을 명확히 정의하기 위한 **참고용 문서**입니다.
실제 의사결정은 다음 요소들을 종합적으로 고려하여 이루어집니다:

- **데이터 민감도** (위 7단계)
- **프롬프트 유형** (정의/접근/통계)
- **사용자 직급 및 권한**
- **프롬프트 내용 분석** (의도, 범위, 대상)
- **맥락 위험도** (컨텍스트)
- **회사 AI 정책 준수 여부** (개인정보 외부 AI 금지)

따라서 같은 민감도의 데이터라도 프롬프트 내용과 사용자 상황에 따라 
의사결정이 **Allow에서 Block**까지 다르게 결정될 수 있습니다.

### 법적 책임
- 고객, 임직원, 설계사 개인정보를 외부 생성형 AI에 입력 = 법령 위반
- 위반 시 최대 **전체 매출액의 3% 과징금 부과** 가능
- 실수로 개인정보가 사용됨을 인지한 경우 **즉시 CISO에 알림**
